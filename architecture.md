# 校园反诈智能问答系统 - 系统架构

## 1. 系统概述

校园反诈智能问答系统是一个基于检索增强生成（RAG）技术的智能防诈助手，专为大学生群体设计。系统通过结合本地向量检索和大模型生成，为用户提供专业、准确的反诈建议。

## 2. 系统架构

### 2.1 整体架构

```
用户输入
  ↓
[本地离线] 向量化（Embedding）
  ↓
[本地离线] 混合检索（向量 + 关键词）
  ↓
[本地离线] 检索相关文档
  ↓
对话历史管理
  ↓
Prompt 构造
  ↓
[网络] 调用 DashScope API
  ↓
[网络] 大模型生成答案
  ↓
用户显示结果
```

### 2.2 核心组件

| 组件 | 技术 | 说明 | 位置 |
|------|------|------|------|
| 前端界面 | Streamlit | 交互式Web应用 | `app.py` |
| RAG系统 | Python | 核心业务逻辑 | `src/rag_system.py` |
| Embedding模型 | BAAI/bge-small-zh-v1.5 | 中文文本向量化 | `src/rag_system.py` |
| 向量存储 | ChromaDB | 本地向量数据库 | `chroma_db/` |
| 混合检索 | RRF (Reciprocal Rank Fusion) | 向量 + 关键词融合 | `hybrid_retriever.py` |
| 大模型 | 通义千问（qwen-turbo） | 答案生成 | `src/rag_system.py` |
| 数据处理 | Pandas, NumPy | 数据加载和处理 | `scripts/build_kg.py` |

## 3. 数据流

### 3.1 知识库构建流程

1. **数据加载**：从 `data/fraud_cases.csv` 和 `data/fraud_policies.txt` 加载数据
2. **数据处理**：对诈骗案例进行结构化处理，生成标准化文档
3. **文本分块**：使用自适应分块算法对文本进行分块
4. **向量化**：使用 Embedding 模型将文本转换为向量
5. **存储**：将向量和元数据存储到 ChromaDB

### 3.2 查询处理流程

1. **问题输入**：用户输入问题
2. **智能判断**：判断是否为非诈骗相关问题
3. **向量化**：将问题转换为向量
4. **检索**：使用混合检索在向量数据库中检索相关文档
5. **对话历史管理**：维护和更新对话历史
6. **Prompt构造**：将检索结果、问题和对话历史构造为Prompt
7. **大模型调用**：调用通义千问生成答案
8. **结果返回**：将答案返回给用户

## 4. 模块设计

### 4.1 RAGSystem 类

**核心功能**：
- 系统初始化（加载模型和向量存储）
- 问题处理（智能判断、检索、生成）
- 模型管理（Embedding模型和大模型）

**主要方法**：
- `__init__()`：初始化系统
- `answer_query(query, history=None)`：处理用户查询，支持对话历史
- `_call_llm(prompt)`：调用大模型
- `_create_retriever()`：创建混合检索器
- `_is_non_fraud_query(query)`：判断是否为非诈骗问题
- `clear_history()`：清空对话历史

### 4.2 HybridRetriever 类

**核心功能**：
- 混合检索（向量检索 + 关键词检索）
- RRF（Reciprocal Rank Fusion）结果融合
- 政策文件软惩罚
- 校园贷专属关键词强化
- 结果去重和排序

**主要方法**：
- `invoke(query, k)`：执行检索
- `_extract_keywords(query)`：提取关键词
- `_calculate_relevance_score()`：计算相关性分数
- `_fuse_rankings()`：融合排序结果

### 4.3 前端模块

**核心功能**：
- 用户界面渲染
- 问题提交和结果显示
- 快速示例功能
- 对话历史管理

**主要组件**：
- 页面布局和样式
- 输入处理
- 结果展示
- 侧边栏设置
- 对话历史显示

## 5. 技术特点

### 5.1 本地离线检索

- **优势**：保护用户隐私，不需要网络连接
- **实现**：使用 ChromaDB 本地向量存储
- **性能**：检索速度 < 100ms

### 5.2 智能问题识别

- **优势**：能够区分诈骗相关和非诈骗相关问题
- **实现**：基于关键词匹配和模式识别
- **效果**：提高用户体验，避免对无关问题的误判

### 5.3 自适应分块

- **优势**：根据内容类型自动调整分块策略
- **实现**：对不同类型内容使用不同的分块参数
- **效果**：提高检索准确性和上下文完整性

### 5.4 内存优化

- **优势**：减少内存使用，避免系统卡顿
- **实现**：批处理、内存释放、增量处理
- **效果**：支持在普通配置的机器上运行

## 9. 部署方案

### 9.1 本地部署

- **环境要求**：Python 3.10+, 4GB+ 内存
- **安装步骤**：

  2. 配置 API 密钥
