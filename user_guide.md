# 校园反诈智能问答系统 - 用户指南

## 系统概述

校园反诈智能问答系统是基于检索增强生成（RAG）技术的智能防诈助手，专为大学生群体设计。系统能够识别常见的校园诈骗类型，并提供专业的风险评估和防范建议。

### 系统功能

- 🚨 **风险评估**：识别是否为诈骗行为，评估风险等级
- ⚠️ **警示信号**：列出诈骗行为的典型特征
- 🛡️ **防范建议**：提供具体的防范步骤和措施
- 📞 **应急处理**：遭遇诈骗后的处理方法
- ⚖️ **法律知识**：相关法律后果说明

### 支持的诈骗类型

- 兼职刷单诈骗
- 假冒教务处诈骗
- 校园贷诈骗
- 虚假奖学金诈骗
- 虚拟人设交友诈骗
- 游戏账号诈骗
- 冒充辅导员诈骗
- 虚假实习诈骗
- 快递理赔诈骗
- 网络直播诈骗
- AI换脸诈骗
- 电信诈骗
- 网络购物诈骗
- 投资理财诈骗

## 安装指南

### 系统要求

- Python 3.10 或更高版本
- 4GB 以上内存
- 网络连接（用于大模型 API 调用）
- 本地模型文件：BAAI/bge-small-zh-v1.5

### 安装步骤

1. **克隆或下载项目**
   ```bash
   cd anti-fraud-rag-complete
   ```
2. **创建虚拟环境（推荐）**

   Windows:
   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```
   Mac/Linux:
   ```bash
   python -m venv venv
   source venv/bin/activate
   ```
3. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```
4. **下载并设置本地模型**

   **方案一：手动下载模型文件（最可靠）**
   1. 在浏览器打开这个地址（hf-mirror 国内镜像，无需魔法）：
      ```
      https://hf-mirror.com/BAAI/bge-small-zh-v1.5/tree/main
      ```
   2. 下载以下 5 个文件，全部保存到同一个文件夹，比如 `D:\models\bge-small-zh-v1.5\`：
      - config.json
      - tokenizer\_config.json
      - tokenizer.json
      - special\_tokens\_map.json
      - model.safetensors （约 93MB，这是最大的那个）
5. **配置 API 密钥和模型路径**
   ```bash
   # 复制示例文件
   copy .env.example .env
   ```
   编辑 `.env` 文件，添加你的 DashScope API 密钥和模型路径：
   ```
   DASHSCOPE_API_KEY=sk-your-key-here
   EMBEDDING_MODEL_PATH=D:\models\bge-small-zh-v1.5
   ```
   获取 API 密钥：<https://dashscope.console.aliyun.com/>
6. **构建知识库**
   ```bash
   python scripts/build_kg.py
   ```
7. **运行应用**
   ```bash
   streamlit run app.py
   ```
   浏览器会自动打开应用，访问 <http://localhost:8501>

## 常见问题

### Q: 为什么需要 API 密钥？

A: API 密钥用于调用阿里云通义千问大模型，实现高质量的答案生成。检索部分完全本地离线，不需要密钥。

### Q: 用户数据会被保存吗？

A: 不会。系统不存储任何用户问题或个人信息。检索完全在本地进行，只有 API 调用需要网络。

### Q: 支持离线使用吗？

A: 支持部分离线。检索部分完全离线，大模型生成需要网络。可以修改代码只使用检索结果，实现完全离线。

### Q: 如何下载本地模型？

A: 推荐手动下载模型文件（最可靠）：

1. 在浏览器打开：<https://hf-mirror.com/BAAI/bge-small-zh-v1.5/tree/main>
2. 下载以下 5 个文件到同一文件夹（如 D:\models\bge-small-zh-v1.5\）：
   - config.json
   - tokenizer\_config.json
   - tokenizer.json
   - special\_tokens\_map.json
   - model.safetensors

### Q: 如何配置模型路径？

A: 在 .env 文件中设置 EMBEDDING\_MODEL\_PATH 环境变量：

```
EMBEDDING_MODEL_PATH=D:\models\bge-small-zh-v1.5
```

这是系统默认的模型路径，如果你使用不同的路径，需要相应修改。

### Q: 如何添加更多知识库？

A: 编辑 `data/fraud_cases_optimized.csv`，添加新的诈骗案例，然后运行：

```bash
python scripts/build_kg.py
```

### Q: 支持其他语言吗？

A: 目前仅支持中文。可以通过翻译数据来支持其他语言。

### Q: 系统响应时间如何？

A: 总响应时间通常在 1-3 秒之间，具体取决于网络速度和问题复杂度。

## 故障排除

### 1. 无法加载 RAG 系统

**症状**：应用启动时显示"无法加载 RAG 系统"错误

**解决方法**：

- 确保已运行 `python scripts/build_kg.py` 构建知识库
- 确保 `chroma_db/` 目录存在且不为空
- 检查 `requirements.txt` 中的依赖是否已安装

### 2. API 调用失败

**症状**：系统返回"API 密钥未配置"或"API 调用失败"错误

**解决方法**：

- 确保 `.env` 文件中已正确配置 `DASHSCOPE_API_KEY`
- 确保 API 密钥格式正确（以 sk- 开头）
- 检查网络连接是否正常

### 3. 内存使用过高

**症状**：运行 `scripts/build_kg.py` 时系统卡顿

**解决方法**：

- 确保系统内存充足（至少 4GB）
- 关闭其他占用内存的应用程序
- 系统已优化为批处理模式，会自动管理内存

### 4. 检索结果不准确

**症状**：系统返回的结果与问题相关性不高

**解决方法**：

- 尝试更详细地描述问题
- 调整侧边栏中的"检索结果数量"参数
- 确保知识库已正确构建
